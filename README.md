# RadAssist PRO

Веб-приложение-ассистент для врачей-рентгенологов.
Объединяет клинические калькуляторы (TI-RADS, Fleischner, BI-RADS, Lung-RADS), трекер динамики образований и генератор структурированных заключений.

---

## Стек технологий

| Слой | Технологии |
|------|-----------|
| Фронтенд | React 18 + TypeScript, Tailwind CSS, Zustand, Recharts, React Hook Form + Zod |
| Бэкенд | Node.js + Fastify, PostgreSQL 16, Redis, Supabase Auth |
| Инфраструктура | Docker, Kubernetes, S3/MinIO |

---

## Структура проекта

```
radiology/
├── frontend/          # React-приложение (Vite)
├── backend/           # Node.js + Fastify API
├── shared/            # Общие TypeScript-типы и схемы
└── docker-compose.yml
```

### Этап 1 — Бэкенд: основа (MVP)

#### 1.1 База данных
- [ ] Проектирование схемы: `users`, `patients`, `lesions`, `measurements`, `reports`, `scale_results`, `subscriptions`
- [ ] Написание миграций (node-postgres / Knex migrations)
- [ ] Настройка подключения к PostgreSQL и Redis

#### 1.2 Аутентификация
- [ ] Интеграция Supabase Auth (JWT, refresh-ротация, 15 мин TTL)
- [ ] Эндпоинты: `POST /auth/register`, `POST /auth/login`, `POST /auth/refresh`
- [ ] Middleware для проверки JWT на всех защищённых маршрутах
- [ ] Двухфакторная аутентификация (TOTP)
- [ ] Ролевая модель доступа (RBAC): `doctor`, `head`, `admin`

#### 1.3 API — Калькуляторы шкал
- [ ] `GET /scales/:type/config` — конфигурация шкалы (для PWA-кеша)
- [ ] `POST /scales/:type/calculate` — серверная валидация расчёта
- [ ] JSON Schema валидация входных данных на каждом эндпоинте

#### 1.4 API — Трекер образований
- [ ] `GET /lesions`, `POST /lesions` — список и создание
- [ ] `GET /lesions/:id`, `PUT /lesions/:id`, `DELETE /lesions/:id`
- [ ] `POST /lesions/:id/measurements` — добавление измерения
- [ ] `GET /lesions/:id/dynamics` — расчёт VDT, данные для графика

#### 1.5 API — Заключения и профиль
- [ ] `POST /reports/generate` — генерация структурированного заключения
- [ ] `GET /reports`, `GET /reports/:id`, `DELETE /reports/:id`
- [ ] `GET /users/profile`, `PUT /users/profile`, `PUT /users/settings`

#### 1.6 Экспорт документов (Pro-тариф)
- [ ] `POST /reports/:id/export/docx` — генерация DOCX (библиотека `docx`)
- [ ] `POST /reports/:id/export/pdf` — генерация PDF (Puppeteer headless)
- [ ] Загрузка файлов в S3/MinIO, возврат временной ссылки

#### 1.7 Безопасность
- [ ] TLS 1.3 на всех эндпоинтах
- [ ] Псевдонимизация пациентов (без ФИО, только год рождения + псевдоним)
- [ ] Шифрование чувствительных полей в БД (AES-256)
- [ ] Аудит-лог всех операций с данными пациентов
- [ ] CORS: разрешены только домены приложения

- [ ] Кнопка «Сохранить в трекер» (открывает форму привязки к пациенту/образованию)

#### 2.5 Модуль трекера образований

- [ ] Автоматический расчёт объёма по формуле эллипсоида: `V = (π/6) × a × b × c`

- [ ] Цветовая индикация строк: жёлтый (рост > 10%), зелёный (уменьшение > 10%), серый (стабильно)
- [ ] Подсветка VDT < 400 дней красным
